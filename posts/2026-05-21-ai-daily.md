### AI 做数学

- **OpenAI 模型推翻离散几何中心猜想** — HN 658 pts, 467 comments  今日最大焦点
  - **事件**: OpenAI 宣布其模型成功证明了一个离散几何领域的核心猜想是*错误的*——不是验证已知定理，而是发现了一个反例推翻了长期存在的猜想。
  - **为什么重要**: 在此之前，AI 在数学中的角色主要是"辅助验证已有证明"或"搜索已知结论"。这是少有的 AI 主动发现反例推翻人类猜想的案例，属于"AI 做原创数学"的标志性事件。
  - **方法**: OpenAI 未完全公开技术细节（博客已发布），但从描述看涉及将几何猜想转化为可计算的搜索空间+模型驱动的反例搜索+形式化验证。
  - **影响**: ① 数学界：离散几何领域的研究方向将被重新审视；② AI 界：AI for Math 从"辅助证明"进入"原创发现"新阶段；③ OpenAI 品牌：在 Anthropic 风头正劲时，用数学突破打出差异化。
  - **局限**: 单一案例，泛化到其他数学领域的能力未验证；反例搜索 vs 构造性证明是两种不同能力。

### Hermes Agent 学习循环深度解析 — Dev.to 9♡

- **核心机制**: 多数 Agent "记不住"。Hermes 的学习循环（Learning Loop）通过将失败经验转化为持久化的 Skill 文件，实现跨会话的渐进改进。
- **与 FORGE 的呼应**: 昨天 arXiv 的 FORGE 论文（群体广播进化记忆）和今天的 Hermes 实战文章，从学术和工程两端指向同一方向——Agent 记忆不应只靠权重更新，文本化经验+反思循环是更实际的路线。
- **一句话评价**: Skill 文件是 Agent 的"长期记忆"，这一模式正在成为共识。

### Per-User OAuth for AI Agents — Dev.to 6♡

- **核心问题**: AI Agent 读取你的 Slack、起草你的邮件、管理你的日历——但它应该以*你的*身份还是以*服务*的身份做这些？传统 OAuth 是 service-to-service，Agent 需要的是 user-to-service 的受控代理。
- **关键设计**: 用户授权→Agent 代理执行→操作审计→权限可随时撤销。
- **与 MCP 安全联动**: 昨天的 MCP on-device 文章（Dev.to 27♡）和今天的安全讨论指向同一个问题——AI Agent 的安全边界在哪里设置、怎么执行。

## 三、科技圈动态

### GitHub 遭 VS Code 恶意扩展供应链攻击，3,800 仓库被攻破 — HN 471 pts, 156 comments

- **事件**: GitHub 确认一名员工设备通过恶意 VS Code 扩展被攻陷，导致约 3,800 个仓库的访问凭证泄露。这是近年最大的开发者工具链供应链攻击之一。
- **影响分析**:
  - **直接威胁**: 受影响仓库的代码、Secrets、CI/CD 凭证可能已泄露。GitHub 已通知受影响用户并强制轮转令牌。
  - **开发者信任危机**: VS Code 扩展的权限模型再次被质疑——扩展可以访问文件系统、终端、Git 凭证，但安全审计严重不足。
  - **MCP 安全关联**（Dev.to 5♡ 文章直接指出）: 如果 VS Code 扩展可以攻破 GitHub，MCP Server 作为 AI Agent 的"扩展"，权限更大、审计更少，将是下一个攻击面。
- **行业应对**: 最小权限的 VS Code 扩展沙箱、MCP Server 的 OAuth 级授权（呼应上文 Per-User OAuth）、"不给 AI SSH 权限"（呼应之前的 homelab doctor）成为安全共识。

### Google "向 Web 宣战" 争议升级 — HN 284 pts, 160 comments

- **背景**: 昨天 Google I/O 宣布搜索框大改——AI 直接回答问题，用户不再需要点击链接进入网站。
- **今日发酵**: 一篇名为"Google Declaring War on the Web"的博客引爆 HN——核心论点是 Google 的新搜索模式截断了网站流量来源，"AI 答案扼杀了内容生态的源头"。
- **两方观点**:
  - 批评方: Google 从"索引 Web"变成"替代 Web"，内容创作者失去流量激励，长尾内容将枯萎。
  - 支持方: 用户想要的是答案不是链接，AI 搜索提升了信息获取效率；内容创作者应适应新分发渠道。
- **关联**: 与昨天的 OpenAI+Google SynthID 合作形成对比——在内容溯源上合作，在流量生态上角力。
