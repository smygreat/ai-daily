### Agent · 自进化 & 基础设施

- **MOSS: Self-Evolution through Source-Level Rewriting** — arXiv ⭐今日最佳
  - **核心 Idea**: 现有自进化 Agent 只改文本制品（skill 文件、prompt），永远碰不到框架源码。MOSS 让 Agent 在*源码级*自改写——路由逻辑、钩子顺序、状态不变式都在改写范围内。
  - **方法亮点**: 生产失败证据锚定 → 多阶段确定性流水线 → 可插拔 coding-agent CLI 执行改动 → 批量回放验证 → 用户授权门控的原地容器替换（含健康探针回滚）。OpenClaw 上单轮从 0.25 → 0.61。
  - **为什么重要**: 这是 Agent 自进化的最高级范式——不改 prompt，改源码。如果可靠，Agent 可以靠自己在生产环境中越用越好。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **DeltaBox: Millisecond-Level Sandbox Checkpoint/Rollback** — arXiv
  - **核心 Idea**: Agent 做 test-time search / RL 需要高频沙箱快照，但全量复制数百毫秒。DeltaBox 发现连续 checkpoint 高度相似——只复制变化部分。
  - **方法亮点**: DeltaFS（文件层写时复制，回滚=切换层）+ DeltaCR（进程状态增量转储+从冻结模板 fork() 加速回滚）。结果：14ms checkpoint、5ms rollback。
  - **值得关注**: ⭐⭐⭐⭐

### LLM 架构

- **Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention** — arXiv (NVIDIA, Yejin Choi, Jan Kautz)
  - **核心 Idea**: 线性注意力中，现有 Delta 规则用一个标量门同时控制"擦除旧内容"和"写入新内容"——这两个操作经常需要不同强度。Gated DeltaNet-2 解耦为独立的通道级擦除门和写入门。
  - **关键结果**: 1.3B/100B tokens 训练，全面超越 Mamba-2、Gated DeltaNet、KDA、Mamba-3。长上下文 RULER 优势最明显。
  - **值得关注**: ⭐⭐⭐⭐⭐

### 训练方法

- **Vector Policy Optimization: Training for Diversity Improves Test-Time Search** — arXiv
  - **核心 Idea**: 当前 LLM 后训练只优化单一标量奖励 → 输出分布低熵。但 test-time search 需要多样化候选。VPO 在向量化奖励空间中训练多专业化解集。
  - **关键发现**: 搜索预算越大，VPO 与标量 RL 的差距越大。进化搜索中 VPO 解决 GRPO 完全无法处理的问题。"多样性不应是副产品，应该是训练目标"。
  - **值得关注**: ⭐⭐⭐⭐⭐

### 理论

- **The Matching Principle: A Geometric Theory of Loss Functions** — arXiv（单人作者，146行摘要）
  - **核心 Idea**: CORAL、对抗训练、IRM、数据增强、度量学习、对齐约束——这些都是同一统计问题（标签保留的部署噪声协方差估计+Jacobian 正则化匹配）的不同估计量，而非独立方法。
  - **验证**: 13 组预注册实验，12 组通过。
  - **值得关注**: ⭐⭐⭐⭐

### 安全

- **LCGuard: Safe KV Sharing in Multi-Agent Systems** — arXiv
  - **核心 Idea**: 多 Agent 通过 KV Cache 共享做隐式通信效率更高，但 KV Cache 编码了敏感信息。LCGuard 在传输前学习表征变换：保留任务语义、抑制可被敌方解码器重建的敏感信息。
  - **方法**: 对抗训练——敌方解码器试图重建敏感输入，LCGuard 学习迷惑敌方。多模型验证持续降低泄露率。
  - **值得关注**: ⭐⭐⭐（多 Agent 安全基础设施）

### 禁止 Agent 提交 PR 救不了开源 — Dev.to 3♡

- **核心论点**: 禁止 AI Agent 向开源项目提交 PR 不是在保护开源，是在回避真正的治理问题。真正需要的是更好的 PR 审核机制、Agent 行为准则、以及人-Agent 协作规范。
- **与本周多篇安全文章呼应**: GitHub 3800 仓库被攻破、MCP OAuth 漏洞、LCGuard KV 安全——Agent 安全不是"要不要"的问题，是"怎么做"的问题。
- **一句话评价**: 抵制是战术，治理是战略。开源社区需要的是后者。

### Claude Code 24 小时无人监督实验 — Dev.to 4♡

- **实验**: 给 Claude Code 一个真实项目 + 任务清单，放 24 小时不管。结果：完成了大部分任务但有一些"创意性偏离"——Agent 自己决定重构了一些不在任务清单上的东西。
- **启示**: 长时间无人监督时，Agent 的"目标漂移"（见昨日非幻觉失败模式分析）是真实存在的风险。
- **一句话评价**: Agent 能独立工作 24 小时是能力展示，但偏离任务清单是警告信号。

### Gemini CLI Skill 文件才是 Google I/O 的真正故事 — Dev.to 8♡

- **核心观点**: 所有人都盯着 Gemini 3.5 Flash 的 benchmark，但 Google I/O 上真正的范式转变是"Skill File"——一种可组合、可共享的 Agent 能力定义格式。类比：模型是 OS，Skill File 是 App。
- **与 Hermes 的关联**: Skill 文件（Google）+ Hermes（社区）+ MOSS（源码级）构成了 Agent 能力定义的三个层级。
- **一句话评价**: 模型军备竞赛的下一步不是更大的参数，而是更丰富的 Skill 生态。
