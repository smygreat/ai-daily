## 一、论文速递

### LLM 架构 / 推理

- **DashAttention: Differentiable and Adaptive Sparse Hierarchical Attention** — arXiv
  - **核心 Idea**: 现有层次注意力（NSA, InfLLMv2）用硬 top-k 选择 KV 块，既固定了每次选择的块数，又切断了从稀疏层到密集层的梯度流。DashAttention 用 α-entmax 自适应选择可变数量的块，让整个注意力层级全程可微。
  - **方法亮点**: 75% 稀疏度下达到全注意力精度，高稀疏区域碾压 NSA 和 InfLLMv2。Triton 实现的推理速度超过 FlashAttention-3。证明了 DashAttention 的"非发散性"——长上下文建模能力更强。
  - **对比**: vs NSA → DashAttention 解决了 NSA 中 top-k 的不可微性和固定预算问题；vs InfLLMv2 → 更好的 Pareto 前沿。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **Predictable Confabulations: Factual Recall by LLMs Scales with Model Size and Topic Frequency** — arXiv
  - **核心 Idea**: LLM 的"事实幻觉"不是随机的——召回质量遵循 S 型曲线，由模型参数量和训练数据中主题出现频率的对数线性组合决定。38 个模型、8900+ 学术引用验证。
  - **关键发现**: 仅模型大小和主题频率两个变量就解释了 16 个 dense 模型间 60% 的方差（同家族内 74-94%）。与 superposition hypothesis 一致——信号强度随概念频率缩放，噪声底限随模型容量缩放。
  - **实践启示**: 对于高频主题，大模型的事实召回可靠；对于低频/小众知识，即使最大模型也不可靠。不要盲目信任大模型的事实性。
  - **值得关注**: ⭐⭐⭐⭐

### Agent

- **Code as Agent Harness: A Unified Survey** — arXiv（超大规模综述，40+ 作者）
  - **核心 Idea**: 代码已不再是 LLM 的输出目标，而是 Agent 的操作基础。提出"代码作为 Agent 骨架"（Code as Agent Harness）统一框架，把代码在 Agent 系统中的三重角色——接口（连接推理/行动/环境建模）、机制（规划/记忆/工具使用）、扩展（多 Agent 协作）——系统化梳理。
  - **涵盖范围**: 编程助手 → GUI/OS 自动化 → 具身 Agent → 科学发现 → 个性化推荐 → DevOps → 企业工作流。
  - **开放挑战**: 超越最终任务成功的评估、不完全反馈下的验证、无回退式骨架改进、多 Agent 共享状态一致性、安全关键场景的人类监督、多模态环境扩展。
  - **值得关注**: ⭐⭐⭐⭐⭐（如果你在做 Agent，这是必读路线图）

- **ESI-Bench: Towards Embodied Spatial Intelligence — Li Fei-Fei, Yejin Choi 等** — arXiv
  - **核心 Idea**: 真正的空间智能不是被动"看"，而是通过行动主动"发现"——遮挡后的结构、物体的动力学、容器的容纳性。构建了覆盖 10 大类 29 子类的具身空间智能评测。
  - **关键发现**: ① 主动探索远优于被动观察；② 模型能自发涌现空间策略；③ 多数失败不是感知弱，而是"行动失明"——错误行动→错误观察→级联错误；④ 人类会在矛盾证据面前修正信念，模型则在任何证据质量下都过早高置信度地下判断——暴露了元认知鸿沟。
  - **值得关注**: ⭐⭐⭐⭐⭐

### CV / 多模态

- **Vision-OPD: Learning to See Fine Details via On-Policy Self-Distillation** — arXiv
  - **核心 Idea**: 同一个 MLLM 看裁剪区域时能答对细节问题，看全图时却答错——这不是能力问题，是注意力问题。Vision-OPD 让模型自己做自己的老师：裁剪图作 teacher → 全图作 student → 在线策略自蒸馏。
  - **方法亮点**: 不需要外部教师模型、不需要 GT 标签、不需要 reward verifier、推理时也不需要额外工具。在多个细粒度视觉理解 benchmark 上达到甚至超越更大的开源/闭源模型和 Thinking-with-Images Agent。
  - **值得关注**: ⭐⭐⭐⭐

### AI 社会影响 & 伦理

- **What Does the AI Doctor Value? Auditing Pluralism in Clinical Ethics of Language Models** — arXiv (Harvard/Brown)
  - **核心 Idea**: 医学伦理天然是多元主义的——自主、有利、不伤害、公正经常冲突，优秀医生会在尊重患者价值观的前提下灵活权衡。AI 医生呢？研究发现前沿模型之间存在医生级别的伦理价值差异（Overton 多元主义），但*单个模型*的决策在多次采样和语义变体下几乎确定——缺乏分布性多元主义。
  - **关键发现**: 大多数模型的价值优先级在医生间自然变异范围内，但部分模型显著低估患者自主权。一个 LLM 不加调整就部署，会在每个患者身上放大其单一伦理立场——用"部署单一文化"取代临床多元主义。
  - **值得关注**: ⭐⭐⭐⭐

### 具身 / 机器人

- **DexHoldem: Playing Texas Hold'em with Dexterous Embodied System** — arXiv
  - **核心 Idea**: 用 ShadowHand 打德州扑克来评测具身系统——不是单一技能测试，而是感知→决策→执行→场景维护的完整闭环。提供 1470 条遥操作示范。
  - **关键发现**: π₀.₅ 任务完成率最高（61.2%），Opus 4.7 严格场景级感知最好（34.3%），GPT 5.5 字段级最准（66.8%）。三者之间的差距说明：孤立的视觉子能力和完整的状态恢复之间有巨大鸿沟。
  - **值得关注**: ⭐⭐⭐（具身 Agent 评测设计思路有启发性）

## 二、Agent & 应用

### Forge: Guardrails 让 8B 模型从 53% → 99% — HN 225 pts
- **是什么**: 开源 Agent 护栏框架，在 agentic 任务上将 8B 模型的成功率从 53% 拉到 99%。
- **优点**: GitHub 开源，轻量级护栏设计，不依赖更大模型的蒸馏。
- **对比**:
  | 维度 | Forge | Reflexion | 传统 fine-tune |
  |------|-------|-----------|---------------|
  | 机制 | 运行时护栏/验证 | 事后反思 | 权重更新 |
  | 成本 | 推理时额外验证 | 推理时反思 | 训练成本 |
  | 提升幅度 | 53→99% (8B) | 增量提升 | 依赖数据和算力 |
- **一句话评价**: "护栏替代微调"是 Agent 落地的重要方向，99% 的数字需要看具体 benchmark，但思路正确。

### Bad Prompt 烧 $40 Claude 预算 — Dev.to 5♡
- **事件**: 多 Agent 循环中一个工具调用卡住→无限重试→18 分钟烧掉 $40。
- **教训**: 单次调用有成本日志，但没有*共享预算上限*。修复方案是原子预算计数器——超 $5 即熔断。
- **一句话评价**: Agent 的容错不是"不要错"，是"错的代价可控"。这是所有 Agent 生产部署的第一步。

### AI 不取代初级程序员，组织架构图才是 — Dev.to 20♡
- **核心观点**: AI 没有"杀死"初级岗位，公司的扁平化设计和培训预算削减才是。两个看似矛盾的论述（"AI 需要经验丰富者操作" vs "AI 替代了初级人力"）其实指向不同层面——个体能力 vs 组织设计。
- **启示**: 不要被"AI 取代人类"的叙事掩盖真实的组织管理问题。
