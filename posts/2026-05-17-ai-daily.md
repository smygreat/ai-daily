## 一、论文速递

### LLM / 推理

- **OpenDeepThink: Parallel Reasoning via Bradley-Terry Aggregation**
  - **核心 Idea**: 不止让模型一条路思考到底，而是同时生成多条推理路径，用 Bradley-Terry 成对比对机制投票选出最优解，然后变异迭代 —— 本质是把"群体智慧"引入推理阶段。
  - **方法亮点**: Gemini 3.1 Pro 在 Codeforces 上 Elo +405 分提升，8 轮迭代约 27 分钟。框架无关模型强弱，无需重训即可迁移。同时发布 CF-73 标注数据集。
  - **局限性**: 主观领域（如创意写作）效果反而下降，赢家通吃机制偏向可客观验证的任务。
  - **对比**: 相较于 OpenAI o1/o3 的"单链深度扩展"，这是"广度扩展 + 投票聚合"路线，与 DeepSeek-R1 的 MCTS 方向有交集但更轻量。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **ATLAS: Agentic or Latent Visual Reasoning? One Word is Enough for Both**
  - **核心 Idea**: 提出"功能 Token"——单个离散 token 既代表 Agent 工具调用又代表隐式视觉推理，避免生成冗长的中间图像，同时保持标准 next-token prediction 训练范式。
  - **方法亮点**: 引入 Latent-Anchored GRPO 解决功能 token 在 RL 训练中的稀疏性问题。在多个视觉推理 benchmark 上效果 SOTA。
  - **局限性**: 功能 token 的语义可解释性依赖后验分析，预定义的操作集限制了开放性场景。
  - **对比**: 相比 SayCan/RT 系列的显式工具调用，ATLAS 的 token-level 融合更优雅；相比 GILL 等隐式方法，它保留了可解释的操作边界。
  - **值得关注**: ⭐⭐⭐⭐

### Agent

- **FutureSim: Replaying World Events to Evaluate Adaptive Agents**
  - **核心 Idea**: 不回测历史数据，而是"重放未来"——用真实新闻按时间顺序喂给 Agent，测试其对新信息的实时适应和预测能力。
  - **方法亮点**: 2026 年 1-3 月 3 个月的 global events 模拟，最强大模型预测准确率仅 25%，多数模型不如随机猜测。发现模型在 long-horizon adaptation、记忆、不确定性推理方面普遍薄弱。
  - **对比**: 相比 MMLU/GSM8K 等静态 benchmark，这是真正考验 Agent「在开放世界中持续适应」的能力，更接近 AGI 评测需求。
  - **值得关注**: ⭐⭐⭐⭐⭐

### CV / 多模态

- **EntityBench: Towards Entity-Consistent Multi-Shot Video Generation**
  - **核心 Idea**: 多镜头视频生成最大的痛点是角色/物体/场景的跨镜头一致性问题。提出 EntityBench（140 集、2491 个镜头），显式追踪每个镜头中实体出现情况。
  - **方法亮点**: 提出 EntityMem——在生成开始前建立"实体记忆库"存储视觉参考。Cohen's d = +2.33 的角色一致性提升。
  - **关键发现**: 跨镜头一致性随镜头间距急剧下降，说明现有方法缺乏长程记忆。
  - **对比**: 相比 VBench/EvalCrafter 等评测，这是第一个系统化评测「长程实体一致性」的 benchmark。
  - **值得关注**: ⭐⭐⭐⭐

- **VGGT-Edit: Feed-forward Native 3D Scene Editing**
  - **核心 Idea**: 不再走"2D 编辑 → 3D 提升"的间接路线，而是在 3D 空间内直接做文本引导的场景编辑。
  - **方法亮点**: 深度同步文本注入确保语义与空间对齐，残差变换头直接预测 3D 几何位移。构建了 DeltaScene 数据集。推理速度接近瞬时。
  - **对比**: 相比 Instruct-NeRF2NeRF 等 2D-lifting 方法，纹理更清晰、多视角一致性更强。
  - **值得关注**: ⭐⭐⭐

- **PDI-Bench: Quantitative Video World Model Evaluation**
  - **核心 Idea**: 给视频世界模型做"几何体检"——通过透视几何残差诊断生成视频的 3D 结构是否合理。三个维度：尺度深度对齐、3D 运动一致性、结构刚性。
  - **关键发现**: 主流视频生成模型在几何一致性上存在系统性缺陷，且这些缺陷被 FVD/IS 等感知指标掩盖。
  - **值得关注**: ⭐⭐⭐（对 Sora/SANA 类方向有重要评估意义）

### RL / 训练方法

- **Shodh-MoE: Eradicating Negative Transfer in Multi-Physics Foundation Models**
  - **核心 Idea**: 同时训练多种物理方程（流体+多孔介质）时，共享参数路径造成梯度冲突。用 Top-1 软语义路由器动态将不同物理域分配到不同专家子网络。
  - **方法亮点**: 训练过程中自动形成领域分岔——开放通道 tokens 全部路由到 Expert 0，多孔介质 tokens 全部路由到 Expert 1。质量守恒的物理可验证误差 ~2.8×10⁻¹⁰。
  - **值得关注**: ⭐⭐⭐（虽偏科学计算，但 MoE 自动分工的思路对通用 AI 训练有启发）

## 二、Agent & 应用

### Zerostack — HN 210 pts, 67 comments
- **是什么**: 纯 Rust 写的类 Unix 编码 Agent，命令行驱动的 AI 编程助手
- **优点**: 极简 Unix 哲学设计，Rust 原生性能，单二进制部署。与终端工作流天然集成。
- **缺点**: 生态极早期，只支持 crates.io 发布，距离 Copilot/Cursor 的 IDE 集成体验还有差距。
- **横向对比**:
  | 维度 | Zerostack | Claude Code | Cursor | Aider |
  |------|-----------|-------------|--------|-------|
  | 架构 | 独立 CLI Agent | 独立 CLI Agent | IDE 插件 | CLI Agent |
  | 语言 | Rust | TypeScript | TypeScript | Python |
  | 成熟度 | 早期 (v1.0) | 成熟 | 成熟 | 中等 |
  | 定位 | Unix 极简哲学 | 全功能 Agent | IDE 集成 | 轻量 AI 结对 |
- **一句话评价**: 有态度的新玩家，Unix 哲学 + Rust 的差异化路线有意思，但需证明能追上头部产品的迭代速度。

### Hermes Agent 7日自进化实验 — Dev.to 19♡
- **是什么**: 开源 Agent 在同一个任务上连续运行 7 天，skill 文件从 Day 1 到 Day 7 面目全非
- **优点**: 唯一开源自主进化型 Agent，7 天自动迭代展示了 self-improvement 的可行性
- **缺点**: 进化方向不可控——"面目全非"可能是进步也可能是 drift
- **横向对比**: vs Manus（闭源通用 Agent）→ 更开放但更不稳定；vs AutoGPT → 更聚焦 task persistence
- **一句话评价**: 开源 Agent 赛道值得关注的实验，但稳定性是落地的最大障碍。

### Gemma 4 MoE vs Dense 对比实验 — Dev.to 9♡
- **是什么**: 在阿拉伯语电商客服场景下，给 Gemma 4 26B (MoE, 4B active) 和 31B (dense) 加了三条 prompt 规则后的对比
- **关键发现**: MoE 变体"打开目录搜索"、dense 变体"停止阅读"——同样 prompt + 不同架构 → 截然相反的行为。说明 MoE 的稀疏激活在特定场景下比 dense 更擅长完成结构化任务。

### PR Review Agent — Dev.to 5♡
- **性质**: Hermes Agent Challenge 投稿，AI 代码审查 Agent
- **痛点**: 现有 AI Code Review 工具（如 CodeRabbit）往往给出泛泛的评论而非深入分析

- **SANA-WM: 2.6B 开源世界模型** — HN 310 pts, 128 comments

  - **适用场景**: 视频生成、物理模拟、具身智能训练的数据增强
  - **同类对比**: 相较于 Sora（闭源）、Stable Video Diffusion（开源但参数更大），SANA-WM 在小参数下实现了有竞争力的质量。值得作为世界模型研究基线。
  - **上手难度**: 中，需 GPU（推荐 A100），但开源代码完整。
