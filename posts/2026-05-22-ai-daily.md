## 一、论文速递

### Agent

- **Agent JIT Compilation for Web Agent Planning** — arXiv ⭐今日最佳
  - **核心 Idea**: 当前 Web Agent（Computer-Use Agent）是一个低效的串行循环——截图→调 LLM→执行→再截图。Agent JIT 把任务描述"编译"成可直接执行的代码，并行化工具调用，像 JIT 编译器一样优化执行计划。
  - **方法亮点**: 三个组件——① JIT-Planner：生成多份代码计划，验证后选最低成本方案；② JIT-Scheduler：基于学习到的延迟分布做蒙特卡洛并行化策略；③ 不变式工具协议：前置/后置条件检查减少错误工具调用。vs Browser-Use → 10.4× 加速 + 28% 准确率提升；vs OpenAI CUA → 2.4× 加速 + 9% 准确率提升。
  - **为什么重要**: 这是 Web Agent 方向的一个范式级改进——从"逐步推理执行"到"编译式并行执行"。如果普及，Web Agent 的延迟将不再是瓶颈。
  - **对比**: vs 传统的 fetch-execute loop → JIT 类比于从解释执行到编译执行，编译时一次性做验证和优化。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **Mem-π: Adaptive Memory through Learning When and What to Generate** — arXiv
  - **核心 Idea**: 现有 Agent 记忆是"检索式"——从经验库中按相似度拉静态记录，经常对不上当前上下文。Mem-π 是"生成式记忆"——用一个独立模型（自己的参数，不和 Agent 共享）实时决定"要不要给建议、给什么建议"。
  - **方法亮点**: 决策-内容解耦的 RL 训练目标——模型学会"不需要时闭嘴，需要时给精准指导"。Web 导航任务上相对提升 30%+。
  - **对比**: vs RAG 类检索记忆 → Mem-π 是 on-demand generation，检索的是"静态快照"，生成的是"上下文定制的动态建议"。与 FORGE（群体广播）和 Hermes（Skill 文件）构成 Agent 记忆的三条路线。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **DeepWeb-Bench: A Hard Deep Research Benchmark** — arXiv
  - **核心 Idea**: 现有 deep research benchmark 太简单，前沿模型都快刷爆了。DeepWeb-Bench 从三个维度加难度——海量证据收集、跨源对齐、长程多步推导。每道题附来源溯源记录。
  - **关键发现**: ① 检索不是瓶颈（检索失败仅占 12-14% 错误，推导+校准失败占 70%+）；② 强模型和弱模型的失败模式质的不同——强模型坏在推导不完整，弱模型坏在幻觉精度；③ 模型在不同领域间真正存在专业分化（跨模型一致性仅 ρ=0.61）。
  - **值得关注**: ⭐⭐⭐⭐

### LLM 训练

- **Quantifying Hyperparameter Transfer: Why μP Works** — arXiv
  - **核心 Idea**: 为什么 μP（Maximal Update parameterization）能实现超参数从小模型到大模型的迁移？通过三项指标系统量化后发现——μP 最大的好处几乎全部来自*最大化 Embedding 层学习率*。
  - **关键发现**: 在 SP（标准参数化）下，Embedding 层学习率是训练不稳定性的瓶颈；把它按宽度放大到与 μP 相同后，训练平滑度显著改善，超参数迁移性大幅提升。Weight decay 提升 scaling law 拟合质量，但在固定 token-per-parameter 设定下损害外推鲁棒性。
  - **实践价值**: 如果你不想切 μP，至少把 embedding LR 调到够大——这是 μP 优势的 80%。
  - **值得关注**: ⭐⭐⭐⭐

### CV / 多模态

- **WikiVQABench: Knowledge-Grounded VQA Benchmark** — arXiv
  - **核心 Idea**: 现有 VQA 大多靠看图就能答（"图里有几只狗"）。现实中的问题是"这个建筑属于哪个建筑流派"——需要外部知识+视觉证据。WikiVQABench 从 Wikipedia 图文对+Wikidata 知识图谱构建，人类审核。
  - **关键发现**: 15 个 VLM（256M-90B）准确率跨度 24.7%-75.6%，说明知识密集型视觉推理是当前 VLM 的显著短板。
  - **值得关注**: ⭐⭐⭐

### RL / 训练方法

- **CARV: Compute-Aware Variance Reduction for Diffusion Teachers** — arXiv
  - **核心 Idea**: 用扩散模型作为"冻结教师"做 text-to-3D、单步蒸馏等下游任务时，计算瓶颈在于 MC 梯度估计的高方差。CARV 通过分层 MC 估计器——把昂贵的上游计算（渲染/仿真）摊销到便宜的扩散噪声重采样上——实现 2-3× 有效计算乘数。
  - **值得关注**: ⭐⭐⭐（text-to-3D 方向的实用优化）

### AI 学术出版

- **AiraXiv: AI-Driven Open-Access Platform for Human and AI Scientists** — arXiv (ICAIS 2025 平台)
  - **核心 Idea**: 当前学术出版已经承载不了论文增长速度（审稿人不够用、投稿量暴涨）。AiraXiv 让人类和 AI 科学家共同作为作者和读者，论文通过持续反馈迭代演进。AI 科学家通过 MCP 协议交互。
  - **信号**: "AI 作为科学家参与学术"不是一个未来设想——AiraXiv 已经上线运行并承办了 ICAIS 2025。
  - **值得关注**: ⭐⭐⭐

### $48K GPU 服务器值不值 — HN 257 pts, 195 comments

### Waymo 暂停亚特兰大服务：无人车反复驶入洪水 — HN 236 pts, 297 comments

- **事件**: Waymo 在亚特兰大的无人出租车在多处积水路段反复驶入，已暂停服务。
- **分析**: 这不是传感器故障——是"常识推理"的缺失。积水深度、水流速度、水下路况这些对人类司机显而易见的危险，对 AV 系统仍是大盲区。回看昨天 ESI-Bench 论文的结论——"行动失明"导致级联错误。
- **信号**: 自动驾驶的 edge case 不是越来越少，而是越来越"常识化"——不是感知不到水，是不知道水多深算危险。
