### RL / 训练方法

- **When Chaos Wins: Adding Noise Improved My Snake AI's Stability** — Dev.to 4♡
  - **核心 Idea**: 在 Rainbow DQN 训练贪吃蛇时，主动加入噪声反而提升了策略的稳定性。这不是新理论，而是一个"反直觉工程发现"——在资源受限的小规模训练中，噪声充当了廉价的正则化和探索机制。
  - **方法亮点**: 逐组件构建 Rainbow DQN（Double DQN → Prioritized Replay → Dueling → Noisy Nets），每一步都有对比消融，方法论扎实。
  - **启示**: 学术界追求去噪，工程界有时需要加噪。小规模 RL 训练中，过拟合比欠拟合更致命。
  - **值得关注**: ⭐⭐（工程经验 > 学术贡献，但方法论对入门 RL 有参考价值）

## 二、Agent & 应用

### Semble — HN 150 pts, 40 comments
- **是什么**: 面向 AI Agent 的代码搜索引擎，比 grep 少用 98% tokens
- **为什么重要**: Claude Code / Copilot 等 Agent 在探索代码库时会反复 grep，token 消耗巨大。Semble 用语义搜索 + 结构化索引替代暴力文本匹配，直击 Agent 工具调用的成本痛点。
- **优点**: 大幅减少 token 消耗 → 降低 API 成本 + 加快响应速度；GitHub 开源 (MinishLab/semble)
- **缺点**: 需要预建索引，实时性不如 grep；小众语言支持待完善
- **横向对比**:
  | 维度 | Semble | grep/ripgrep | Sourcegraph | GitHub Code Search |
  |------|--------|-------------|-------------|-------------------|
  | 定位 | Agent 专用语义搜索 | 通用文本搜索 | 企业代码搜索 | 平台代码搜索 |
  | Token 效率 | 极高 | 极低（全量输出） | 中等 | 中等 |
  | Agent 集成 | 原生设计 | 需包装 | 需 API | 需 API |
  | 成熟度 | 早期 | 极成熟 | 成熟 | 成熟 |
- **一句话评价**: Agent 工具链正在从"拼模型"转向"拼基础设施"，Semble 抓住了正确的切入口。

### CodeGraph — Dev.to 5♡
- **是什么**: 阻止 AI Agent 对同一文件反复 grep 50 次的工具
- **与 Semble 的差异**: Semble 做搜索端优化，CodeGraph 做代码图谱索引——构建代码结构图让 Agent 一次性理解项目拓扑，而非反复试探。
- **一句话评价**: 和 Semble 从不同角度解决同一问题，可互补使用。

### Four LLM Workflows That Actually Survive Production — Dev.to 5♡
- **是什么**: 从生产环境中总结的 4 个真正能活的 LLM 工作流模式
- **核心观点**: 大多数团队死在"试图造魔法助手"上，而真正落地的都是枯燥但可靠的单一工作流。
- **4 个工作流**: ① 分类路由（意图识别→分发）② 结构化抽取（文档→JSON）③ 检索增强生成（RAG with citation）④ 人机协同审核（human-in-the-loop）
- **一句话评价**: 没有 magic，只有 engineering。这篇文章是对 LLMOps 炒作的一剂清醒药。

### AI Agent Loop in Go — Dev.to
- **是什么**: 用 Go 从零构建个人 AI Agent（nevinho），支持 Bash/文件/浏览器工具调用
- **亮点**: Go 语言做 Agent loop 的选择非常规但合理——并发原生、部署单二进制。作者分享了实际踩坑经验（工具超时处理、上下文窗口管理）。
- **一句话评价**: Go + Agent 是值得关注的小众组合，适合需要高并发工具调用的场景。

- **Gemma 4 on i5 CPU** — Dev.to 3♡
  - 在消费级 i5 CPU 上运行 Gemma 4，证明了"不需要 RTX 也能玩 LLM"。
  - **适用场景**: 本地推理、边缘部署、学习和实验。
  - **技巧**: 量化 + CPU 推理优化（llama.cpp / Ollama），推理速度可接受但生成 token/s 仍远低于 GPU。

- **Browser AI + Core Web Vitals 实测** — Dev.to 11♡
  - 在浏览器中直接跑 AI 模型（WebGPU/WebNN）对 Core Web Vitals 的影响量化分析。
  - **关键发现**: 模型加载阶段对 LCP 影响最大，推理阶段对 INP 影响显著。需要 Web Worker 隔离 + 模型缓存策略。
  - **适用场景**: 浏览器端 AI 功能的性能优化参考。
