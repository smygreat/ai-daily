## 一、论文速递

### LLM 架构 / 推理

- **Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention** — arXiv (NVIDIA, Yejin Choi, Jan Kautz)
  - **核心 Idea**: 线性注意力用固定大小的循环状态替代 softmax 注意力的无限缓存。但现有 Delta 规则模型用一个标量门同时控制"擦除旧内容"和"写入新内容"——这两个操作经常需要不同的强度。Gated DeltaNet-2 将它们解耦为独立的通道级擦除门和写入门。
  - **方法亮点**: 1.3B/100B tokens 训练规模下全面超越 Mamba-2、原始 Gated DeltaNet、KDA、Mamba-3。长上下文 RULER needle-in-haystack 任务优势最明显。推导了分块 WY 算法 + 门控反向传播，保持高效并行训练。
  - **对比**: vs Gated DeltaNet → 解耦擦除/写入是核心差异；vs KDA → 两个独立通道门退化为同一标量时的特化版本。NVIDIA 出品，代码开源。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **Vector Policy Optimization: Training for Diversity Improves Test-Time Search** — arXiv
  - **核心 Idea**: 当前 LLM 后训练只优化单一标量奖励，导致输出分布低熵、缺乏多样性。但 test-time search（如 AlphaEvolve）恰恰需要多样化候选才能有效。VPO 显式训练策略在*向量化*奖励空间（多个测试用例/多个人格/多个 reward model）中产生专业化分工的解集。
  - **关键发现**: 随着搜索预算增加，VPO 与标量 RL 基线的差距拉大。在进化搜索中，VPO 模型解决 GRPO 模型完全无法处理的问题。口号是——"多样性不应是副产品，应该是训练目标"。
  - **对比**: vs GRPO → VPO 可视为 GRPO 优势估计器的即插即用替代，但输出的是多专业化的解集而非单一策略。
  - **值得关注**: ⭐⭐⭐⭐⭐

### Agent

- **MOSS: Self-Evolution through Source-Level Rewriting in Autonomous Agent Systems** — arXiv ⭐今日最佳
  - **核心 Idea**: 现有自进化 Agent 只改文本制品（skill 文件、prompt、记忆模式），永远碰不到 Agent 框架本身的代码——路由逻辑、钩子顺序、状态不变式都在源码里。MOSS 让 Agent 在*源码级*自改写，这是图灵完备的、确定性执行的、不受长上下文漂移影响的进化方式。
  - **方法亮点**: 每条进化锚定于生产失败证据的自动批处理→多阶段确定性流水线→可插拔外部 coding-agent CLI 执行改动→批量回放验证→用户授权门控的原地容器替换（含健康探针回滚）。OpenClaw 上单轮无人工干预从 0.25 提升到 0.61。
  - **为什么重要**: 这是 Agent 自进化的最高级范式——不改 prompt，改源码。如果可靠，Agent 可以靠自己在生产环境中越用越好。
  - **值得关注**: ⭐⭐⭐⭐⭐

- **DeltaBox: Millisecond-Level Sandbox Checkpoint/Rollback for AI Agents** — arXiv
  - **核心 Idea**: AI Agent 做 test-time search / RL 时需要高频沙箱快照和回滚，但现有全量复制方案每次数百毫秒到秒级延迟。DeltaBox 发现 Agent 的连续 checkpoint 高度相似——只需要复制变化部分。
  - **方法亮点**: 两个 OS 级抽象——① DeltaFS：文件层分层组织，写时复制，回滚只是切换层；② DeltaCR：进程状态增量转储+从冻结模板进程直接 fork() 加速回滚。结果：14ms checkpoint、5ms rollback。
  - **对比**: 传统全量复制 → 两个数量级的延迟改善。SWE-bench 和 RL 微基准上，DeltaBox 让 Agent 在固定时间预算内探索更多节点。
  - **值得关注**: ⭐⭐⭐⭐

- **LCGuard: Safe KV Sharing in Multi-Agent Systems** — arXiv
  - **核心 Idea**: 多 Agent 系统通过 KV Cache 共享做隐式通信效率更高，但 KV Cache 编码了上下文输入+推理状态+Agent 特有信息——敏感内容可能通过"不透明通道"泄露。LCGuard 在传输前学习表征层变换：保留任务语义、抑制可被敌方解码器重建的敏感信息。
  - **方法**: 对抗训练——敌方解码器试图从 KV Cache 重建敏感输入，LCGuard 学习迷惑敌方的变换。多个模型家族和基准上持续降低重建泄露率和攻击成功率。
  - **值得关注**: ⭐⭐⭐（多 Agent 安全的关键基础设施）

### LLM 理论

- **The Matching Principle: A Geometric Theory of Loss Functions** — arXiv（单人作者，146行摘要）
  - **核心 Idea**: 鲁棒性、域适应、组合泛化、对齐安全等十几个看似无关的领域共享同一统计问题——估计标签保留的部署噪声协方差，然后用 Jacobian 正则化匹配它。CORAL、对抗训练、IRM、数据增强、度量学习、对齐约束都是该协方差的不同估计量，而非独立的方法。
  - **雄心**: 用一套几何理论统一多个独立的鲁棒性研究方向。包含闭式最优证明、13 组预注册实验（12 组通过）。
  - **值得关注**: ⭐⭐⭐⭐（如理论成立，是对鲁棒性研究方向的重大简化和统一）

### AI Agent Failure Modes Beyond Hallucination — Dev.to 14♡

- **四个非幻觉失败模式**: ① 工具调用死循环（Agent 在工具间反复跳转不推进任务）；② 目标漂移（长任务中 Agent 悄悄换了目标）；③ 部分执行（完成 70% 后认为已完成）；④ 权限误用（以更高权限执行了不必要的操作）。
